Information Flow

All signals, mouse clicks and keypresses are passed onto the canvas_t class, which then passes them onto either the drawing_t class or performs the necessary actions.

Points

Points are added on to the screen on left click. On left click the function canvas.left_click(x, y) is called that records the pixel position. It calls _left_click(x,y) by converting (x,y) from screen frame to canvas frame.

Points are drawn by the canvas_t on demand by the point_t class.

Mode

Changing modes only changes how drawing_t responds to the points, and nothing else. o, the debug point mode, line and triangle modes all behave similarly.

Objects

The drawing objects (line and triangles) are treated similarly. They are stored in the order they are created with data about the vertices, the line thickness used to create them, the border and fill color, etc. This makes redrawing an image very simple.

Lines are drawn by drawing the points computed via the line drawing algorithm and Triangles are drawn using the edges.

Canvas

Canvas is just a 3D array of adjustable length and breadth with pixel color data which enables OpenGl to draw them. It contains no further data.

Saving/Loading

Stream operators are overloaded enabling easy input and output of drawing and recreating the drawing very quickly and exactly with no loss of information.

Algorithms

For line drawing, integer version of Bresenham algorithm is used, with flood fill as the default filling algorithm. The thickness of lines is implemented by updating nearby pixels as well while editing a single pixel in canvas_t
